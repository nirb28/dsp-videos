DSP AI JWT Service

A Flask-based JWT token service designed for integration with APISIX gateway. This service provides JWT token generation with flexible authentication methods and support for dynamic claims based on API keys.

Overview

What is JWT Service?

The JWT Service is a centralized authentication service that generates and validates JSON Web Tokens (JWT) for the DSP AI Platform. It supports multiple authentication methods and provides dynamic claims based on API key configurations.

Key Features

  • Multiple Authentication Methods - LDAP and file-based authentication
  • Configurable JWT Tokens - Shared secret with APISIX gateway
  • Flexible API Key Configuration - File-based and inline payload support
  • Dynamic Claims - Function and API-based claim generation
  • Token Refresh - Secure token refresh mechanism
  • Token Decoding - Verify and decode JWT tokens
  • Control Tower Integration - Use manifest configurations

Best Practices

#E3FCEF Security Best Practices
  • Strong Secrets - Use cryptographically secure keys (min 32 characters)
  • Short Expiration - Access tokens expire in 30-60 minutes
  • Refresh Tokens - Use separate long-lived refresh tokens
  • HTTPS Only - Always use encrypted connections
  • Secret Rotation - Regularly rotate signing keys
  • Audit Logging - Log all authentication attempts
  • Rate Limiting - Prevent brute force attacks
 #FFF0B3 Performance Best Practices
  • Token Caching - Cache tokens in Redis to reduce generation
  • Async Claims - Use async for API-based dynamic claims
  • Connection Pooling - Reuse connections for external APIs
  • Claim Caching - Cache expensive dynamic claim results
  • Batch Operations - Batch multiple claim API calls

Troubleshooting

Authentication Failures

Problem: User cannot authenticate

Solutions:

  • Verify username and password are correct
  • Check user exists in users.yaml or LDAP
  • Verify password hash is correct (for file-based)
  • Check LDAP server connectivity (for LDAP auth)
  • Review authentication logs
 Token Validation Errors

Problem: Token rejected by APISIX or services

Solutions:

  • Verify JWT_SECRET_KEY matches across services
  • Check token hasn't expired
  • Validate token format (three parts separated by dots)
  • Ensure algorithm matches (HS256, RS256, etc.)
  • Check issuer and audience claims
 Dynamic Claims Not Working

Problem: Dynamic claims missing or incorrect

Solutions:

  • Check function module and function name are correct
  • Verify API endpoint is accessible
  • Review placeholder substitution in arguments
  • Check response_field path for API claims
  • Review logs for claim generation errors
